    코웨이 지속가능경영

    • 2017년 6월, 7월 작업
    • IE8이상 호환, 반응형 페이지
    • 서브페이지 90% (대부분)
    • 페이지 수가 80개 정도 되는 많은 페이지에서, 반복되는 css를 가이드 페이지로 먼저 만들어 놓고 코딩을 진행하는 방식을 처음 해보았습니다. http://sustainability.coway.co.kr/html/sub01.asp (가이드 페이지 링크)
    • 기본적인 모양에선 필요 css를 클래스 이름만 붙이면 되었기 때문에 훨씬 빠르고 수월하게 코딩을 칠 수 있었습니다. 반응형도 이미 가이드 페이지에서 고려되었기 때문에 반응형 역시 좀 더 빠르게 코딩이 가능 하였습니다. 하물며 기본적 모양에선 코딩을 모르는 디자이너들도 급한 상황에선 코딩을 도와줄 수 있을 정도로 코딩이 간편화되는 효율적인 방법을 배운 것 같습니다.
    • 매일같이 클라이언트의 수정사항을 받으며, 맨 처음 디자인과는 계속해서 달라지는 걸 반응형까지 고려해서 코딩하려니 밤샘 작업도 이뤄지며 몸도 많이 상한 일이기도 합니다. 기술적인 부분에서 배운 것도 많았지만, 무엇보다도 컨디션 조절과 적절한 일의 진행에 대한 중요성을 깊게 느끼게 해준 일이기도 하였습니다.

    케미스토리

    • 2016년 12월부터 2017년 2월까지
    • IE7이상 호환
    • main 레이아웃 + sub60%（메뉴, 스크립트제외)
    • 회사에 들어와서 처음으로 한 작업으로 사용하면 안 되는(하위버전 호환이 안 되는) css구분에 대해 배웠습니다.
    • nth-child()같은 가상선택자 대부분을 쓸 수 없지만 first-child는 쓸 수 있다는 점, opacity 같은 css3 기능은 대부분 하위에서 호환이 안 된다는 점 등을 알게 되었습니다.
    • css reset을 최소화한 상태에서 작업하며 기본 태그들이 margin값을 얼마나 크게 갖고 있는 지에 대해 알게 되었습니다.
